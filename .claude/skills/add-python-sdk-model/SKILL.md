# Add or Update a Python SDK Model

Guide for adding or updating Python SDK models generated from the Kubeflow Trainer CRDs.

## How It Works

Python API models in `api/python_api/kubeflow_trainer_api/models/` are auto-generated from the OpenAPI specification. The generation pipeline is:

1. CRD types defined in `pkg/apis/trainer/v1alpha1/` (Go)
2. `openapi-gen` generates OpenAPI spec from Go types
3. Python models generated from the OpenAPI spec

You never edit Python model files directly - they are overwritten on each generation.

## Steps

### 1. Modify the Go API types

Edit the relevant `*_types.go` file in `pkg/apis/trainer/v1alpha1/`.

For new fields, add proper markers:

```go
// FieldName describes what this field does.
// +optional
FieldName *string `json:"fieldName,omitempty"`
```

The `json` tag determines the Python model field name. Use camelCase in Go, which maps to snake_case in the Python model.

### 2. Run code generation

```bash
make generate
```

This regenerates everything: deepcopy, CRD manifests, clientset, OpenAPI spec, and the Python API models.

### 3. Verify the Python models

Check that your changes appear in the generated Python models:

```bash
# Look for your new field in the generated models
grep -r "field_name" api/python_api/kubeflow_trainer_api/models/
```

### 4. Run Python tests

```bash
make test-python
```

### 5. Verify the full build

```bash
make generate
make fmt
make test
make test-python
```

## Important Notes

- Never edit files under `api/python_api/kubeflow_trainer_api/models/` directly - they are generated and excluded from pre-commit hooks
- The Python API version is tracked in `api/python_api/kubeflow_trainer_api/__init__.py`
- If you add a completely new CRD type, it will automatically get a Python model after `make generate`
- Python models are generated by `hack/python-api/gen-api.sh` (invoked by `make generate`, requires a container runtime); `hack/update-codegen.sh` generates the Go clientset/informers/listers

## Common Mistakes

- Editing generated Python files directly (they get overwritten)
- Forgetting `make generate` after Go type changes
- Mismatched json tags (the json tag name is what appears in the Python model)
- Missing `+optional` marker on optional fields (affects the generated model's required fields)
