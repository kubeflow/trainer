# Add a New CRD Type

Step-by-step guide for adding a new Custom Resource Definition to Kubeflow Trainer.

## Steps

### 1. Define the Go types

Create or edit a `*_types.go` file in `pkg/apis/trainer/v1alpha1/`.

Required code generation markers on the main type:

```go
// +genclient
// +k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object
// +kubebuilder:object:root=true
// +kubebuilder:subresource:status
// +kubebuilder:storageversion
```

For the list type:

```go
// +k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object
// +kubebuilder:object:root=true
```

Use `+optional` on all optional fields. Use [CEL validation](https://kubernetes.io/docs/reference/using-api/cel/) via `+kubebuilder:validation:XValidation` for field-level constraints. See existing types in `trainjob_types.go` and `trainingruntime_types.go` for patterns.

Add the Apache 2.0 boilerplate header from `hack/boilerplate/boilerplate.go.txt`.

### 2. Register types in the scheme

Add your new type and its list type to `addKnownTypes()` in `pkg/apis/trainer/v1alpha1/groupversion_info.go`:

```go
scheme.AddKnownTypes(GroupVersion,
    &YourNewType{},
    &YourNewTypeList{},
    // existing types...
)
```

### 3. Run code generation

```bash
make generate
```

This generates: deepcopy methods, CRD manifests in `manifests/base/crds/`, clientset/informers/listers in `pkg/client/`, OpenAPI spec, and Python API models.

Verify generated files exist and look correct. If `make generate` fails, check your type markers.

### 4. Add webhooks

Create a webhook file in `pkg/webhooks/` following the pattern in `trainjob_webhook.go` or `trainingruntime_webhook.go`:

- Define a `setup*Webhook*()` function
- Implement `CustomDefaulter` (mutating) and/or `CustomValidator` (validating) interfaces
- Register the webhook in `pkg/webhooks/setup.go` by calling your setup function from `Setup()`
- Add webhook manifests - these are generated by `controller-gen` if you use the `// +kubebuilder:webhook:...` markers

### 5. Add a controller

Create a controller file in `pkg/controller/`:

- Implement `reconcile.Reconciler` interface
- Use `ctrl.LoggerFrom(ctx)` for structured logging
- Register in `pkg/controller/setup.go` in the `SetupControllers()` function
- Use the `mgr.GetEventRecorder()` pattern for events

### 6. Add tests

- Unit tests: `*_test.go` files alongside source code
- Use table-driven tests with `map[string]struct` pattern
- Use `utiltesting.NewClientBuilder()` from `pkg/util/testing/` for fake clients
- Webhook tests: follow pattern in `pkg/webhooks/trainjob_webhook_test.go`
- Integration tests: add to `test/integration/` using Ginkgo framework

### 7. Update RBAC

Add RBAC markers on the controller or runtime core file:

```go
// +kubebuilder:rbac:groups=trainer.kubeflow.org,resources=yournewtype,verbs=get;list;watch;create;update;patch;delete
// +kubebuilder:rbac:groups=trainer.kubeflow.org,resources=yournewtype/status,verbs=get;update;patch
```

Then run `make generate` again to regenerate RBAC manifests.

### 8. Verify

```bash
make generate
make fmt
make vet
make golangci-lint
make test
```

## Common Mistakes

- Forgetting to run `make generate` after type changes (breaks CI)
- Missing `+optional` marker on optional fields
- Not adding list type (`YourNewTypeList`) alongside the main type
- Not registering types in `groupversion_info.go`
- Missing boilerplate header (caught by `make verify-boilerplate`)
